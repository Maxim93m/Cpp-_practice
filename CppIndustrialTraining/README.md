# Cpp_practice
С++. Практика

День 1. Основы синтаксиса, переменные, типы, ввод/вывод, условия

Настройка Visual Studio, первая программа.

Переменные: int, float, bool, std::string.

std::cout, std::cin, std::getline, std::setw.

Условные операторы if/else if/else.

Русский язык в консоли (SetConsoleCP, SetConsoleOutputCP).

Проблема буфера ввода и std::cin.ignore().

Итог: написали программу мониторинга компрессора с одиночными параметрами.

День 2. Функции

Что такое функция, зачем нужна.

Синтаксис, параметры, возвращаемое значение.

Передача параметров по значению и по ссылке (int&).

Прототипы функций.

Разбиение кода на логические блоки.

Итог: рефакторинг программы — вынесли ввод, проверки, вывод в отдельные функции. Код стал модульным и читаемым.

День 3. Массивы и векторы

Проблема: как хранить много однотипных данных (например, 10 датчиков температуры).

Статические массивы (float arr[5]).

Динамические массивы — std::vector.

Добавление элементов: push_back.

Перебор: цикл for по индексу и range-based for.

Практика: расширение программы — теперь можно ввести любое количество датчиков температуры, все проверяются в цикле.

Скоро: сдаём День 3 и идём дальше.

День 4. Файловый ввод/вывод

Запись в файл (std::ofstream) — логирование событий, аварий.

Чтение из файла (std::ifstream) — загрузка конфигураций, уставок.

Формирование отчётов.

Зачем: в промышленности нужно сохранять историю работы и настраивать оборудование через файлы.

День 5. Структуры и классы (введение в ООП)

struct — группировка связанных данных (например, один компрессор: имя, температура, давление…).

Массив/вектор структур — несколько компрессоров.

Понятие класса, инкапсуляция, методы.

Результат: сможем создавать полноценные объекты оборудования.

День 6. Указатели и динамическая память

Указатели, ссылки, new/delete.

Умные указатели (std::unique_ptr, std::shared_ptr).

Необходимо для embedded-разработки и эффективного управления памятью.

День 7–8. Итоговый проект

Сбор всего изученного:

Объекты оборудования (классы).

Множество датчиков (векторы).

Логирование в файл.

Чтение конфигурации.

Консольный интерфейс.

Результат: программа, готовая для портфолио.
